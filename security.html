Security Layers

How We Protect Your Account

EstateIQ uses a defense-in-depth security model with multiple overlapping layers of protection.

1
Perimeter Security (WAF)
Web Application Firewall blocks SQL injection, XSS attacks, DDoS, and malicious bot traffic before they reach our servers.
2
Transport Security (TLS 1.3)
All communication between browser and server is encrypted using TLS 1.3. HTTPS enforced sitewide with HSTS headers.
3
Authentication (JWT + 2FA)
Passwords hashed with bcrypt. Short-lived JWT tokens with refresh rotation. Two-factor authentication via OTP mandatory for agents.
4
Authorization (RBAC)
Role-Based Access Control. Buyers, sellers, agents, and admins each have strictly scoped permissions. No privilege escalation possible.
5
Data Encryption (AES-256)
Sensitive data (PAN, Aadhaar, bank details) encrypted at rest using AES-256. Database backups also encrypted and stored separately.

Security Measures Implemented

🔐

Password Security & Lockout

Passwords hashed via bcrypt. Strict login rate limiting prevents brute-force attacks by locking accounts for 1 minute after 3 failed attempts. Checking against known breaches.

🛡️

CSRF & XSS Protection

All forms include CSRF tokens. Content Security Policy (CSP) headers prevent cross-site scripting. Input sanitization on all user-submitted data before storage or rendering.

CSRF tokensCSP headersHelmet.js
⏱️

Session Management

JWT access tokens expire in 15 minutes; refresh tokens in 7 days with rotation. Suspicious login from new device triggers re-authentication. Concurrent session limits enforced.

Bot & Fraud Detection

Google reCAPTCHA v3 on all forms. AI-powered fraud detection flags suspicious listing patterns (fake properties) and unusual user behavior. IP rate limiting on all API endpoints.

Data Privacy & Compliance

Compliant with India's Personal Data Protection Bill and GDPR. Aadhaar/PAN data masked in UI. Users can request data export or deletion. Audit logs for all data access.

RERA Verification

All property listings cross-verified against RERA database via API. Agents undergo KYC before onboarding. Fake or misleading listings are auto-flagged and removed within 24 hrs.

🔍 Security Audits

We conduct quarterly third-party penetration testing and annual security audits. All discovered vulnerabilities are patched within 48 hours. Bug bounty program available for responsible disclosure.

💾 Data Backup & Recovery

Automated daily backups stored in geo-redundant locations. RTO (Recovery Time Objective) of less than 4 hours. All backups encrypted with AES-256 and tested monthly.

📞 Incident Response

24/7 security operations center monitors for anomalies. Defined incident response plan with escalation matrix. Affected users notified within 72 hours in case of any breach — as required by law.

👤 Agent KYC Process

All agents verified with PAN + Aadhaar via Digilocker. RERA registration number mandatory. Background check run before activation. Ongoing monitoring for suspicious listing activity.
